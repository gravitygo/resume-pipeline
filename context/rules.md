# Resume Generation Rules

## Markdown Structure

- Use `#` only for candidate name (once, at top)
- Use `##` for major sections
- Use `###` for sub-entries (job roles, education entries, projects)
- Use `-` for bullet points
- No other heading levels allowed (H4+ not permitted)
- No inline HTML or styling

## Required Sections

1. **Summary** - 2-3 sentence professional overview
2. **Experience** - Job history with achievements
3. **Skills** - Technical and professional skills

## Optional Sections

- **Contact** - Email, phone, location
- **Links** - LinkedIn, GitHub, portfolio, website
- **Education** - Academic background
- **Projects** - Notable projects with descriptions
- **Certifications** - Professional certifications
- **Languages** - Language proficiencies
- **Volunteer Experience** - Volunteer work and community involvement
- **Awards** - Professional awards and recognition

## Content Guidelines

- Focus on achievements over responsibilities
- Use quantifiable metrics when possible
- Keep bullet points concise and impactful
- Use action verbs to start bullet points
- Avoid generic or vague statements

## Section Formats

### Experience Format

```
### [Job Title] — [Company]
*[Location] | [Start Date] – [End Date]*

- [Achievement with measurable impact]
- [Key responsibility or accomplishment]
```

### Education Format

```
### [Degree] in [Field] — [Institution]
*[Location] | [Start Date] – [End Date]*

- GPA: [GPA] (if notable)
- Honors: [Honors/Awards]
```

### Project Format

```
### [Project Name]
*[Technologies Used]*

- [Description of project and impact]
```

### Certification Format

```
- [Certification Name] — [Issuer] ([Date])
```

## Quality Standards

- Professional tone throughout
- Consistent formatting
- No spelling or grammar errors
- ATS-friendly structure
- Dates in consistent format (e.g., "Jan 2020" or "2020-01")
